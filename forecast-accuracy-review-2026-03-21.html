Root Cause Hypothesis: The model was trained on Sep 2025 – Mar 6, 2026 data, which was dominated by fall/winter volumes. March 2026 has seen an early spring surge — Houston highs reaching 80–84°F with dry conditions — driving volume 50–100% above the model's learned winter patterns. The current model's feature weights are anchored to lower-volume periods and lack exposure to warm-weather spring volumes.

Action Taken: Training data updated with Mar 16–20 actuals (15 new rows). Fresh weather history fetched from Open-Meteo. Model backup created (models_dict_backup_2026-03-21.pkl). Retraining execution was attempted but deferred to next available compute window due to environment constraints. Priority: execute retrain.py at the next scheduled run window.
Recommendations
1. Execute Model Retraining Immediately

Training data has been updated. Run python retrain.py at the next available window. The new model should show improved warm-weather calibration.

2. Investigate Wednesday Mar 18 Anomaly

The Mar 18 briefing predicted TX160=145 and TX190=120 — far below even the model's already-low baseline predictions for that week. This suggests a possible data input issue (e.g., stale lag features) in the forecast pipeline for that specific run.

3. Add Seasonal Decomposition Feature

The model lacks explicit seasonal awareness. Consider adding a "week_of_year" feature or a sinusoidal annual cycle feature to help the model learn that March volumes are structurally higher than January volumes, independent of weather.

4. Monitor UT426 — Performing Well

UT426 improved from 30.1% to 25.4% MAPE, outperforming its 32% baseline. The Utah market seems to be behaving more predictably. Continue monitoring but no action needed.

5. Consider Interim Bias Correction

Until the retrained model is deployed, apply a +40% additive correction to TX160/TX190 forecasts and +10% to UT426. This simple bias adjustment would have reduced fleet MAPE from ~36% to ~15% this week.
