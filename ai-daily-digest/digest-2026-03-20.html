Stripe & Tempo Launch Machine Payments Protocol — AI Agents Can Now Make Autonomous Payments

Stripe Blog / Fortune / PYMNTS / The Block · Mar 18 · HIGH

Stripe and crypto startup Tempo launched the Machine Payments Protocol (MPP), an open standard that enables AI agents to make programmatic payments — in both fiat (cards, BNPL) and stablecoins via Shared Payment Tokens (SPTs). Visa contributed to the specification, developing the standards for letting agents pay with credit and debit cards. Businesses can accept MPP payments with a few lines of code using Stripe's PaymentIntents API. Early adopters include Browserbase (agents pay per headless browser session), PostalForm (agents pay to send physical mail), and Prospect Butcher Co. (agents order sandwiches for human pickup). Tempo's mainnet also went live, providing the blockchain settlement layer for agent-to-agent stablecoin transactions.

Why this matters for you: This is the missing economic layer for agentic AI. Until now, agents could use tools and call APIs, but they couldn't pay for things. MPP changes that — your agents can now autonomously purchase services, pay for API calls, settle invoices, and transact with other agents. With Stripe and Visa backing the standard, this isn't a startup experiment — it's production-grade payment infrastructure. If you're building agent workflows that interact with paid services (SaaS APIs, cloud compute, vendor payments), MPP provides the rails. The stablecoin support through Tempo adds a crypto settlement option for agent-to-agent microtransactions that would be uneconomical with traditional card rails.

Perplexity CTO Says MCP Consumes 40–50% of Context Windows — Company Moving Away from Protocol

Ask 2026 Conference / The New Stack · Mar 11 · MEDIUM

At the Ask 2026 conference on March 11, Perplexity CTO Denis Yarats revealed that his company is moving away from MCP internally, citing two fundamental problems: MCP tool descriptions consume 40–50% of available context windows before agents do any actual work, and authentication flows create friction when connecting to multiple services. The criticism comes as MCP has crossed 97 million monthly SDK downloads and been adopted by every major AI provider. Meanwhile, the MCP project published its 2026 roadmap on March 9, with lead maintainer David Soria Parra identifying four focus areas: scaling Streamable HTTP transport, closing lifecycle gaps in the Tasks primitive, building enterprise readiness features, and publishing a standard metadata format for registries.

COEY / Geeky Gadgets / Latent Space / MLQ.ai · Mar 17 · HIGH

Anthropic launched Dispatch as a research preview within Cowork, enabling users to send instructions to Claude from the mobile app while the AI executes tasks on the desktop application in the background. Dispatch creates a single, persistent conversation thread between your phone and Claude Desktop, allowing you to assign tasks like preparing reports, organizing files, and running multi-step workflows without being at your computer. The feature maintains continuity within a single conversation — you can resume tasks, refine drafts, and provide follow-up instructions without restarting workflows. Why this matters for you: Dispatch fundamentally changes how you interact with Claude for business operations. Instead of sitting at your desk waiting for multi-step workflows to complete, you can fire off instructions from your phone — "generate this week's wash report," "update the P&L dashboard," "research competitor pricing" — and Claude handles it on your desktop while you're on-site or in meetings. Combined with the scheduled tasks feature already in Cowork, this creates a pattern where Claude is always working in the background: scheduled tasks handle recurring work, and Dispatch handles ad-hoc requests from wherever you are.

Security & Governance

Entro Security Launches AGA — First Enterprise Identity Governance Layer for AI Agents

Help Net Security / GlobeNewsWire / Entro Security · Mar 18 · HIGH

Entro Security launched Agentic Governance & Administration (AGA) at RSA Conference 2026, the first purpose-built identity and access governance platform for AI agents. AGA uses EDR integrations to discover AI clients and local agent runtimes on workstations, integrates natively with agent foundries (AWS Bedrock, Copilot Studio), and provides MCP activity visibility with policy enforcement. Core capabilities include discovery of all agents and their non-human identities, permissions mapping, policy controls for sanctioned MCP targets, audit trails of allowed and blocked activity, and AI-focused controls to reduce sensitive data exposure. The platform applies traditional IAM fundamentals — inventory, ownership, least privilege, auditability — to the new challenge of governing autonomous AI agents across the enterprise.

Why this matters for you: Entro's AGA is the identity layer that enterprises need before they'll deploy agents at scale. While SurePath (covered yesterday) governs what MCP tools agents can access, Entro governs the agents themselves — who created them, what identities they use, what permissions they have, and what they're actually doing. Together, these products signal that enterprise AI agent deployment is moving from "can we build it?" to "can we govern it?" If you're deploying agents or skills that interact with enterprise systems, expect customers to require agent identity governance as a prerequisite, just as they require SSO and RBAC for human users today.

Visa Scales Agentic Commerce Through Stripe MPP Collaboration — Card Networks Enter the Agent Economy

PYMNTS / CoinDesk / Fortune · Mar 18 · HIGH

Visa contributed directly to the Machine Payments Protocol specification, developing the standards for AI agents to pay using credit and debit cards through the existing card network infrastructure. This marks the first time a major card network has formally entered the AI agent economy, providing the trust and fraud-protection layer that enterprise buyers will require before letting agents make autonomous purchases. The collaboration means MPP transactions can flow through Visa's existing merchant network — agents don't need merchants to adopt new payment rails to transact. Combined with Stripe's fiat and Tempo's stablecoin support, MPP now covers the full spectrum of payment methods an agent might need: cards, BNPL, and crypto.

Why this matters for you: Visa's involvement transforms MPP from a developer tool into enterprise-grade payment infrastructure. When your agents need to purchase supplies, pay vendors, or settle invoices autonomously, they can now do so through the same Visa network your business already uses — with the same fraud protections, dispute resolution, and compliance frameworks. This is particularly significant for operations-heavy businesses: imagine agents that autonomously reorder cleaning chemicals when inventory is low, pay utility bills on schedule, or purchase equipment parts when maintenance is flagged. The payment bottleneck for autonomous agent operations just disappeared.
