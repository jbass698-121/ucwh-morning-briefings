OpenAI Ships GPT-5.4 — 1.05M Context Window, 33% Fewer Factual Errors

Releasebot / LLM Stats / Build Fast with AI · Mar 5-22 · HIGH

OpenAI released GPT-5.4 on March 5 with a 1.05 million-token context window and three variants: Standard, Thinking, and Pro. The model delivers 33% fewer individual factual errors than GPT-5.2, with API pricing starting at $2.50 per million input tokens. GPT-5.4 mini is now rolling out to Free and Go users via the Thinking feature, while GPT-5 Thinking mini will be retired in 30 days. The 1M+ context window puts GPT-5.4 in direct competition with Claude Opus 4.6's 1M context, making million-token workflows a standard expectation rather than a premium feature.

Why this matters for you: Million-token context is now table stakes across the two leading model providers. For your daily data pull and report generation pipeline, this means you can feed entire week-long datasets — wash counts, coupon summaries, weather data, financial records — into a single context window without chunking. The $2.50/M input token pricing makes this economically viable for production automation. The 33% factual error reduction is equally important: fewer hallucinations in financial reporting and operational analysis means less manual validation overhead.

NVIDIA Nemotron 3 Super — 120B MoE Model Built for Multi-Agent Workflows

Build Fast with AI / NVIDIA GTC · Mar 11 · HIGH

NVIDIA announced Nemotron 3 Super at GTC on March 11 — a 120-billion-total-parameter hybrid Mixture-of-Experts model with only 12 billion active parameters per forward pass. The architecture is specifically designed for complex multi-agent applications including software development, cybersecurity triaging, and agentic workflows. By activating just 10% of its parameters per inference call, Nemotron 3 Super offers the reasoning depth of a 120B model at the compute cost of a 12B model, making it viable for high-throughput agent systems that need to run many parallel tool calls without GPU bottlenecks.

Why this matters for you: A 120B-quality model that runs at 12B cost is a breakthrough for agent infrastructure economics. If you're running multi-step automation — daily data pulls, report generation, validation checks — each step currently consumes full-model inference. A MoE architecture like Nemotron 3 Super could let you run 10x more concurrent agent tasks at the same compute budget. Watch for API availability; if NVIDIA makes this accessible via NIM or a cloud endpoint, it could become a cost-effective backbone for high-frequency automation tasks.

LTX 2.3 — 22B Parameter Model Generates Synchronized Video and Audio in One Pass

Renovate QR / Lightricks · Mar 2026 · MEDIUM

Lightricks released LTX 2.3, a 22-billion-parameter Diffusion Transformer model that generates synchronized video and audio in a single forward pass. Perplexity CTO: We're Moving Away from MCP — Context Window Bloat Is the Problem

DEV Community / Ask 2026 Conference · Mar 11 · HIGH

At the Ask 2026 conference on March 11, Perplexity CTO Denis Yarats revealed his company is moving away from MCP internally, citing two fundamental problems: MCP tool descriptions consume 40-50% of available context windows before agents do any actual work, and authentication flows create friction when connecting to multiple services. This is a notable contrarian signal against MCP's explosive adoption — the protocol crossed 97 million monthly SDK downloads in February — and raises legitimate architectural questions about whether a tool-description-heavy protocol is sustainable as agent systems scale to dozens of connected services.

Why this matters for you: Perplexity's critique is worth internalizing even if you're committed to MCP. If you're connecting 10+ MCP servers for your automation pipeline — Slack, Supabase, GitHub, browser tools — those tool descriptions could be eating 40-50% of your context window before Claude even starts reasoning about your data. The practical takeaway: audit how many MCP tools are loaded in each session, disable tools you're not actively using, and watch for the MCP roadmap's metadata format improvements that could compress tool descriptions. This is an optimization problem, not a reason to abandon the protocol.

SurePath AI Ships MCP Policy Controls — Enterprise Security Gates for Agent Tool Access

DEV Community / SurePath AI · Mar 12 · MEDIUM
